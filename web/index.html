How the synthesizers work

Every synthesizer below does the same two things in a different way: fit (learn a model of the real table(s)) and sample (draw brand-new rows from that model, never copies, always freshly generated). The two statistical models learn distributions and correlations directly, so they're fast and their output is easy to reason about; the neural models learn a network that produces rows, which can capture more complex, non-linear relationships at the cost of a training loop.

Gaussian Copula

Single-table · statistical · fastest
  1. Fit each column's own shape. Learn a univariate distribution (Beta, Gamma, Gaussian, truncated, or an empirical curve) that best matches each column alone, ignoring every other column.
  2. Convert real values to percentiles. Using that per-column curve, turn every value into "where does this fall in the distribution": a number between 0 and 1.
  3. Convert percentiles to normal scores. Pass each percentile through the inverse standard-normal curve so every column now looks like the same bell curve, with row-to-row relationships preserved.
  4. Learn the correlation matrix. Now that every column sits on the same scale, measure how they move together (e.g. age ↔ income).
  5. Sample new correlated points. Draw fresh rows from a multivariate normal distribution shaped by that correlation matrix.
  6. Invert back to real units. normal score → percentile → each column's own inverse distribution → a realistic value in its original scale.
Best for: a clean single table where you mainly need correct per-column shapes plus linear/rank correlation preserved, near-instantly and with no training loop. Trade-off: only captures linear/rank relationships; a 3-way interaction between columns can slip through.

HMA (Hierarchical Modeling Algorithm)

Multi-table · statistical · preserves keys
  1. Walk the table tree from the leaves up. Starting at child tables, augment each parent row with summary statistics of its children: how many children it has, and the mean/covariance of their numeric columns.
  2. Fit a Gaussian Copula per table, with those extra "child-shape" columns baked in, so a parent's copula model implicitly encodes what its children usually look like.
  3. Recurse up multiple levels. Grandparents inherit shape statistics from parents (which already encode their own children), all the way to the root table.
  4. Sample top-down. Generate the root table first, including its sampled "child-shape" parameters.
  5. Generate children per parent. For each sampled parent row, use its own sampled child-shape parameters to decide how many child rows to create, and draw them already stamped with that parent's key.
  6. Recurse down for grandchildren, and so on, until every table is generated.
Best for: multiple linked tables where foreign-key relationships must hold, because every child row is generated already carrying a valid parent key, so referential integrity holds by construction, not by luck. Trade-off: more moving parts than a single copula fit, so it's slower and the dead-zone during "Modeling Tables" can take a while on wide schemas.

CTGAN

Single-table · neural (GAN) · captures non-linear structure
  1. Preprocess numeric columns. Fit a Gaussian-mixture model per numeric column and represent each value as (which "mode" it belongs to, how far into that mode); this handles lumpy, multi-modal columns a single bell curve can't.
  2. Preprocess categorical columns. One-hot encode them, and rebalance sampling ("training-by-sampling") so rare categories are shown to the model often enough to be learned, not drowned out by the dominant ones.
  3. Train two competing networks. A Generator turns random noise (plus a condition on one column's value) into a fake row; a Critic tries to tell real rows from the Generator's fake ones.
  4. Adversarial training loop. Every epoch the Critic gets better at spotting fakes and the Generator gets better at fooling it, repeated for many epochs until the Generator's rows are statistically indistinguishable.
  5. Sample. After training, discard the Critic; feed the Generator fresh random noise (plus conditions) to produce as many new rows as you want.
Best for: tables with skewed, multi-modal numeric columns or complex non-linear joint patterns a copula can't see. Trade-off: needs real training time (epochs), and is less predictable/interpretable than a statistical model.

TVAE

Single-table · neural (VAE) · steadier training
  1. Same preprocessing as CTGAN: mode-specific normalization for numeric columns, one-hot for categorical.
  2. Encode. A neural Encoder compresses each real row into a small latent vector (a mean and spread in a compressed space).
  3. Decode. A neural Decoder tries to reconstruct the original row from that latent vector.
  4. Train on two goals at once: minimize reconstruction error (the rebuilt row should match the original) while keeping the latent space close to a clean standard-normal shape, so it's easy to sample from later.
  5. Sample. After training, skip the Encoder entirely: draw a random latent vector from a standard normal and run it through the Decoder to get a new row.
Best for: similar cases to CTGAN, but trains faster and more stably (no adversarial min-max game, so no mode collapse). Trade-off: can produce slightly softer, blurrier distribution edges than CTGAN's sharper mode fidelity.

CopulaGAN

Single-table · hybrid (copula + GAN)
  1. CDF-transform numeric columns. Like Gaussian Copula's percentile step, convert each numeric column to a uniform 0-1 scale using its own empirical distribution first, taming skew and long tails before anything else sees the data.
  2. Hand off to CTGAN. Run the same mode-specific normalization and conditional Generator/Critic adversarial training as CTGAN, but on these pre-tamed columns.
  3. Sample via the trained Generator, exactly as in CTGAN.
  4. Invert the CDF transform to convert each generated uniform value back to the original column's real scale.
Best for: a middle ground: the calibrated, well-behaved marginals of a copula, plus more of the GAN's ability to capture complex joint structure than a pure Gaussian Copula gets alone. Trade-off: still a full GAN training loop under the hood, so it's as slow as CTGAN.
